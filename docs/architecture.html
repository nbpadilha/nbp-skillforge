nbp-forge · architecture

Agent skills, built from reusable bricks

Write a step once, reuse it everywhere, and let a gate guarantee nothing ever drifts. The generated file is standard and self-contained — portable across Claude Code, Codex, Cursor.

01The idea

A skill is a recipe that points to bricks instead of copying them. A deterministic build assembles the final file your agent reads. The brick is the single source of truth — edit it once, every skill that uses it is current on the next build.

🧱 bricks/ (shared pieces) 📄 recipes/ (one per skill, with includes) → ⚙️ forge build → ✅ <out>/<skill>.md (generated)

02The problem it kills

Before — copy-paste rot

The same step pasted into 5–7 skills. Fix one, forget the rest. Versions diverge silently; nobody knows which is right; reviewing means hunting every copy.

After — one source, gate-enforced

The step lives in one brick. Edit it → all skills update on build. A drift-gate blocks any generated file that was hand-edited or fell out of sync.

03Variation is a parameter, never a copy

Two skills, the same brick, different inputs:

<!-- include: run-dir | skill=fix; flags=--prefix fix --track -->     # in fix
<!-- include: run-dir | skill=feature; flags=--track -->            # in feature
        → {{skill}} / {{flags}} substituted inside the brick
The golden rule: variation between skills is a parameter the recipe passes — never a modified copy of the brick.

04Safe lifecycle

Skills are generated, so you never hand-edit the output — you manage them through the forge. Removal is safe by reference count.

newscaffold a new recipe, then build
renamerename a skill (regenerate, drop the stale output)
removesoft-delete the recipe + only the bricks that skill owns; shared bricks stay (you're told which and why)
restorebring a removed skill (and its bricks) back from _archive/
gcfind / archive orphan bricks (used by nobody)

Removing a skill never deletes a shared brick: a brick used by exactly one skill is owned by it; a brick used by several belongs to none and is never touched. Removed items are soft-deleted to _archive/ (versioned, recoverable). Set deletePolicy: "hard" for permanent deletes.

05Glossary

skillAn automated playbook an AI agent follows to do a job.
brickA reusable step kept in one place, shared by many skills.
recipeA skill's source: local content + includes pointing to bricks.
driftWhen copies of the same step stop being equal over time — what this kills.
gateThe automated check that blocks any output out of sync with its recipe.
